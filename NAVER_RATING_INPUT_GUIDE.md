## 입력 절차

### 1단계: Admin Dashboard 접속
```
URL: http://localhost:5000/admin/restaurants
인증: Replit Auth로 로그인 필요
```

### 2단계: 레스토랑 검색 및 편집
1. 검색창에서 레스토랑명 입력
2. 해당 레스토랑 행의 "편집" 버튼 클릭
3. 편집 대화상자 열림

### 3단계: 네이버 플레이스에서 평점 확인
1. 네이버 검색: `네이버 플레이스 [레스토랑명] [주소 키워드]`
2. 예시: `네이버 플레이스 깃뜰 마포구`
3. 평점(별점 5점 만점)과 리뷰 수 확인

### 4단계: Admin UI에서 입력
**"평점 정보 (네이버)" 섹션 찾기**
- **네이버 평점 (0-5)**: 예) 4.5
- **네이버 리뷰 수**: 예) 1234

### 5단계: 저장 및 확인
1. "저장" 버튼 클릭
2. 자동으로 인기지수 재계산됨
3. 메인 페이지에서 등급 변경 확인 가능

## 인기지수 계산 공식 (5점 척도)

### 듀얼 소스 시스템
```
총점 = 네이버 점수(0-2.5) + 구글 점수(0-2.5)

네이버 점수 = (네이버 평점 / 5.0) × 2.5
구글 점수 = (구글 평점 / 5.0) × 2.5
```

### 등급 체계 (6단계)
| 점수 범위 | 등급 | 설명 |
|----------|------|------|
| 4.5 - 5.0 | ★★★★★ | 전설의 맛집 (Legendary) |
| 4.0 - 4.49 | ★★★★ | 최고의 맛집 (Exceptional) |
| 3.5 - 3.99 | ★★★ | 훌륭한 맛집 (Excellent) |
| 3.0 - 3.49 | ★★ | 좋은 맛집 (Very Good) |
| 2.0 - 2.99 | ★ | 괜찮은 맛집 (Good) |
| 0 - 1.99 | - | 신규 (New) |

### 예시 계산
**케이스 1: 깃뜰 (가정)**
- 구글 평점: 5.0 (3,815 리뷰) → 구글 점수 = (5.0/5.0) × 2.5 = 2.50
- 네이버 평점: 4.8 (2,500 리뷰) → 네이버 점수 = (4.8/5.0) × 2.5 = 2.40
- **총 인기지수: 4.90 ★★★★★ (전설의 맛집)**

**케이스 2: 태초갈비 (가정)**
- 구글 평점: 5.0 (2,636 리뷰) → 구글 점수 = 2.50
- 네이버 평점: 4.6 (1,800 리뷰) → 네이버 점수 = 2.30
- **총 인기지수: 4.80 ★★★★★ (전설의 맛집)**

## 데이터 검증

### 인기지수 재계산 스크립트
```bash
npx tsx server/scripts/recalculatePopularityScores.ts
```

### SQL 쿼리로 검증
```sql
-- 전설의 맛집 등급 레스토랑 확인
SELECT 
  name,
  naver_rating,
  google_rating,
  popularity_score,
  CASE
    WHEN popularity_score >= 4.5 THEN '★★★★★ 전설의 맛집'
    WHEN popularity_score >= 4.0 THEN '★★★★ 최고의 맛집'
    WHEN popularity_score >= 3.5 THEN '★★★ 훌륭한 맛집'
    WHEN popularity_score >= 3.0 THEN '★★ 좋은 맛집'
    WHEN popularity_score >= 2.0 THEN '★ 괜찮은 맛집'
    ELSE '신규'
  END as grade
FROM restaurants
WHERE popularity_score >= 4.5
ORDER BY popularity_score DESC;
```
